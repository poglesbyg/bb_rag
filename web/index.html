RAG shouldn't need a server.
bb_rag doesn't have one.

bb_rag ingests any file — .txt, .md, .pdf, .docx, or anything else that's plain text — chunks it by sentence, and answers questions from it with Claude, Ollama, or Hugging Face. No vector database, no framework: just one binary and a local index file.

$ git clone https://github.com/poglesbyg/bb_rag && cd bb_rag && cargo install --path .

or cargo run -- query "..." from inside the repo — no install needed

How it works

any file ── .txt · .md · .pdf · .docx · flat text, chunked by sentence
↓
claude · ollama · huggingface ── embedded (or skipped → TF-IDF)
↓
index.bin ── bincode-encoded, one local file
↓
query · chat — cosine-ranked context, grounded answer

What it does

Three providers
Claude, Ollama, or Hugging Face — swap with --provider, mix embedding and generation independently.
Any file
.txt, .md, .pdf, .docx, or any flat text file. No extension allowlist; unreadable files are skipped, not fatal.
Sentence-aware chunking
Packs whole sentences under a size budget with overlap. Never cuts mid-sentence.
Streaming chat
A REPL with live token streaming and multi-turn memory, via Ollama.
No server
One binary, one local index.bin file. No vector database to run or manage.
TF-IDF fallback
No embeddings API? Claude falls back to in-process TF-IDF automatically — ingest stays fully offline.

Get started

1
Clone & install
$ git clone https://github.com/poglesbyg/bb_rag && cd bb_rag && cargo install --path .
2
Set your API key
$ cp .env.example .env
3
Ingest your docs
$ bb_rag ingest docs/
4
Ask a question
$ bb_rag query "What's in these docs?"

No API key? Use --provider ollama — free, private, runs on your machine.
